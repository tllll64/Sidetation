# Sidetation 项目说明书

> 像设计师一样在真实网页里**直接拖动、拉伸、调整**页面元素，再把改动一键导出成结构化反馈，交给 AI 编码助手（Claude Code / Cursor）落实到源码，或直接导出可用的 CSS。

---

## 一、问题是什么

在网页从「想法」到「上线」的过程中，视觉调整的沟通一直是效率黑洞：

1. **设计与代码之间隔着一层翻译。**
   设计师提出「这个按钮往右挪一点、卡片再宽 40px」，需要靠截图圈注、文字描述来传达。描述不精确，开发反复对齐，来回好几轮。

2. **给 AI 编码助手描述视觉需求太笨拙。**
   现在很多人用 Claude Code、Cursor 改页面。但 AI 看不到你脑子里的布局。你只能用文字「往左一点」「间距大一点」，或者贴张截图——**既不精确，AI 也难以复现你想要的确切数值**。

3. **非技术人员改不动真实页面。**
   设计师、产品经理有直观的调整想法，但没法在真实运行的页面上动手验证，只能靠嘴说、靠画。

**一句话概括问题：视觉上的修改意图，缺少一种「精确、直观、机器可读」的表达方式。**

---

## 二、价值是什么

Sidetation 把「用文字描述改动」升级为「**在真实页面上直接操纵**」，带来三层价值：

| 对象 | 价值 |
| --- | --- |
| **用 AI 编码的开发者** | 拖完直接复制 Markdown，粘给 Claude Code / Cursor——里面有稳定选择器、精确位移量、改动前后的盒模型，AI 拿到的是**数据而非模糊描述**，一次到位。 |
| **前端开发者** | 不必反复改 CSS 试参数，在页面上拖到满意，直接复制 CSS 覆盖样式贴回项目。 |
| **设计师 / 产品经理** | 无需写代码，在真实页面上直接把想法「摆出来」，导出结果就是可落地的需求。 |

核心理念致敬 [agentation](https://github.com/benjitaylor/agentation)：**不替你生成代码，只生成精确的上下文**——但把「文字标注」升级成了「直接操纵」。

## 三、如何解决问题

### 工作流

```
① 注入 Sidetation  →  ② 点选 / 拖拽 / 拉伸元素  →  ③ 导出改动
                                                     ├─ Markdown → 交给 AI 编码助手
                                                     └─ CSS      → 直接贴回项目
```

### 三种注入方式（适配不同场景）

1. **集成到自己的项目**（仅开发环境）
   ```ts
   import { init } from 'sidetation';
   if (import.meta.env.DEV) init();
   ```

2. **Chrome 插件**（推荐给团队）
   `npm run build` 后加载 `extension/` 目录，点工具栏图标即可在任意页面开关编辑模式。仅申请 `activeTab` 权限，**不受页面 CSP 限制**。

3. **控制台粘贴 / 书签小工具**
   构建后在 `dist/` 生成 `console-paste.js` 和 `bookmarklet.txt`，可在任意网站临时注入。

> ⚠️ 在**第三方网站**上注入时，导出的选择器无法对应你的源码，此时应使用「复制 CSS」而非 Markdown。

### 关键设计（为什么它可靠）

- **不破坏页面结构**：移动用 `transform: translate` 实现，不打乱文档流；编辑只写内联样式并保存原值，撤销即完整还原。
- **UI 完全隔离**：工具栏、选框、参考线等全部渲染在 **Shadow DOM** 里，不污染宿主页面样式。
- **选择器足够稳定**：导出时自动过滤 CSS-in-JS 生成的哈希类名，提高选择器在源码中的可对应性。
- **导出对 AI 友好**：Markdown 会提示 AI「用合理的布局属性去实现，而不是照抄 transform」，避免 AI 生成不健康的定位代码。

## 四、现有能力

### 编辑与操作
- **悬浮工具栏**：开始/完成编辑、修改列表、重置、复制 Markdown / CSS。
- **点选元素**：hover 高亮、点击选中；点子元素下钻，`Alt+点击` 上钻到父级。
- **多选批量操作**：`Shift+点击` 加入/移出多选；拖任意已选元素整体移动、`Delete` 批量删除（均记为一次撤销）。
- **对齐与分布面板**：选中 ≥2 个元素出现对齐（左/水平居中/右/上/垂直居中/下）；≥3 个元素支持横向/纵向等距分布。
- **拖拽移动**：Figma 式吸附——自动吸附兄弟/父元素的边缘与中线，带参考线，阈值可配置（默认 6px）。
- **拉伸缩放**：8 个方向手柄，`Shift` 锁定宽高比，自动处理 `content-box` 边框补偿。

### 属性面板（Figma 风格，选中即出现）
- **Auto layout**：Flow（none/column/row/wrap）、W/H、九宫格对齐、Gap、Padding、Clip content。
- **Appearance**：不透明度、圆角。
- **Fill / Stroke**：背景色 + 透明度、描边色 + 粗细。

### 快捷键（对齐 Figma 习惯）
- `Delete` 删除、`⌘Z / ⌘⇧Z` 撤销重做。
- `Enter / Shift+Enter` 下钻/上钻，`Tab / Shift+Tab` 兄弟元素切换。
- 方向键 1px 微调（`Shift` ×10），`Esc` 取消/退出。

### 历史与持久化
- **手势级 undo/redo 栈**：同字段连续修改自动合并为一步；面板可单条还原或全部重置。
- **改动快照缩略图**：每条记录带「优化前 → 优化后」DOM 缩略图（原生 SVG `foreignObject` 实现，零第三方依赖）。
- **会话持久化**：改动自动存入 `localStorage`（按页面路径归档），刷新后自动恢复；选择器失效的记录以 toast 提示未恢复数量。

### 导出格式
- **Markdown（给 AI agent）**：稳定选择器 + 位移量 + before/after 盒模型 + 落地提示。
- **CSS**：可直接粘贴的覆盖样式（transform / width / height）。

## 五、技术特性

- **Vanilla TypeScript，零运行时依赖，框架无关**——可用于 React / Vue / Angular 等任意项目。
- **构建工具 Vite**，打包 ES / UMD 双格式到 `dist/`。
- **性能目标**：初始化 < 100ms，拖拽 ≥ 60fps。
- **兼容现代浏览器**：Chrome / Firefox / Safari / Edge。
- **不收集任何用户数据**。

### 快速开始

```bash
npm install
npm run dev      # 打开本地演示沙盒
npm run build    # 类型检查 + 打包，并生成插件/注入产物
```
