# Sidetation 使用教程（零基础保姆版）

> 这份教程写给**完全没有编程和安装经验**的同事。跟着一步步做就行，遇到不懂的名词看文末「名词解释」。
> 全程大约需要 30 分钟，只有第一次需要装东西，之后每次用只要 3 步。
>
> 如果任何一步的画面和教程说的不一样，**先别继续**，截图发给管理员。

---

## 你会用它做什么

在网页上像拖拼图一样，用鼠标把按钮、图片、文字挪个位置、改个大小，然后点一个按钮，电脑里的 AI 助手就能自动帮你把这些改动写进代码。你**不需要懂代码**。

---

## 名词先扫盲（读一遍就行，不用背）

- **终端**：一个黑色/白色的输入框窗口，用来打字让电脑执行命令。像微信聊天框，只不过对话对象是电脑。
- **命令**：在终端里输入的一行字，回车后电脑就照做。
- **项目文件夹**：Sidetation 这个工具的所有文件放的地方。
- **管理员**：帮你把这个工具准备好的那位同事（通常是给你发这份教程的人）。

---

# 第一部分：只做一次的安装

## 步骤 1：安装 Node.js（电脑的"发动机"）

1. 打开浏览器，访问：**https://nodejs.org**
2. 点击页面上带 **LTS** 字样的绿色按钮下载（LTS = 稳定版）
3. 下载完双击安装包，一路点「继续 / 同意 / 安装」，用默认选项即可
4. 装完就行，不用打开任何东西

## 步骤 2：拿到项目文件夹

**这一步通常管理员已经帮你放好了**（比如放在桌面或某个共享盘）。请先问管理员：
> 「Sidetation 的项目文件夹在我电脑的哪里？」

拿到后，记住它的位置（例如「桌面上的 Sidetation 文件夹」）。

> 如果管理员让你自己下载：他会给你一个下载链接或压缩包，下载后**解压**，得到一个名叫 `Sidetation` 的文件夹，放到桌面即可。

## 步骤 3：打开终端

1. 按键盘 **`Command（⌘）+ 空格`**，弹出搜索框
2. 输入 **`终端`** 或 **`Terminal`**，回车
3. 出现一个可以打字的窗口，就是终端

## 步骤 4：进入项目文件夹（用拖拽，不用打字）

这是最容易出错的一步，我们用「拖拽」偷懒，避免打错字：

1. 在终端里先输入 **`cd`**，然后**打一个空格**（`cd` 后面一定要有空格）
2. 用鼠标把桌面上的 **`Sidetation` 文件夹**，直接**拖进终端窗口**
3. 终端里会自动出现这个文件夹的完整路径，像这样：
   ```
   cd /Users/你的名字/Desktop/Sidetation
   ```
4. 按**回车**

> ✅ 成功的标志：终端那一行的开头，变成了显示在 `Sidetation` 里面的样子。

## 步骤 5：安装工具的"零件"（要装两次）

**第一次**，直接复制下面这行，粘贴进终端，回车，等它跑完（出现新的一行可输入即完成）：
```bash
npm install
```

**第二次**，进入 mcp 子文件夹再装一次。复制粘贴下面**两行**（一起粘贴，回车）：
```bash
cd mcp
npm install
```

装完后，回到上一层文件夹，粘贴这行回车：
```bash
cd ..
```

> ⚠️ 很多人漏了「第二次」的安装，导致后面点同步会失败。请务必装两次。
> ✅ 成功标志：每次都出现类似 `added XX packages` 的字样，没有一大片红色 `error`。

# 第二部分：每次使用都要做（3 步）

> 下面要开**两个终端窗口**，一个跑网页，一个跑"传话员"，两个都要一直开着。

## 步骤 6：启动网页（第 1 个终端）

在刚才的终端里（确认已经在 `Sidetation` 文件夹里），粘贴回车：
```bash
npm run dev
```

✅ 成功标志：出现一行带网址的字，长这样：
```
➜  Local:   http://localhost:5173/
```
**这个终端窗口不要关**，让它一直开着。

## 步骤 7：启动"传话员"（第 2 个终端）

1. 按 **`Command（⌘）+ N`** 或点终端菜单「Shell → 新建窗口」，开一个**新的终端窗口**
2. 重复【步骤 4】：输入 `cd` 空格，把 `Sidetation` 文件夹拖进来，回车
3. 粘贴回车下面两行（进入 mcp 文件夹并启动）：
   ```bash
   cd mcp
   npm run receiver
   ```

✅ 成功标志：出现类似「listening / 8787」的字样，窗口停在那里不动。**同样不要关**。

## 步骤 8：在 CodeBuddy 里打开网页并使用

1. 打开 **CodeBuddy**
2. 在内置预览里访问：**http://localhost:5173/**
3. 页面上会出现一个小工具栏，点「**开始编辑**」
4. 用鼠标点选一个元素（比如一个按钮），拖动它、拉大拉小
5. 改满意后，点工具栏的「**同步 MCP**」
6. 看到提示「**已同步 N 处修改**」就成功了
7. 在 CodeBuddy 对话框里输入这句话，回车：
   > 用 sidetation 读取我刚在页面上做的修改，帮我落实到源码里。

AI 就会自动把你的改动写进代码。完成！

# 遇到问题怎么办（白话版）

| 你看到的情况 | 是什么原因 | 怎么办 |
|---|---|---|
| 点「同步 MCP」提示**同步失败 / 服务未启动** | 第 2 个终端（传话员）没开或关掉了 | 回到【步骤 7】重新启动，别关窗口 |
| 输入命令后一大片**红色报错** | 多半是没装好零件 | 回到【步骤 5】，确认**装了两次** |
| `npm run dev` 说 **command not found** | Node.js 没装好 | 回到【步骤 1】重装 Node.js，装完**关掉终端重新开** |
| CodeBuddy 说**读不到修改** | 还没在网页上点过同步 | 先在网页点「同步 MCP」看到成功提示，再回对话 |
| 预览里按 **Ctrl+Z / Delete 没反应** | 快捷键被编辑器占用了 | 改用工具栏上的按钮操作，鼠标拖拽不受影响 |
| 找不到 `Sidetation` 文件夹 | 不知道放哪了 | 问管理员要文件夹位置 |

# 快速回顾（贴在显示器旁边）

**每次使用：**
1. 终端 1：`cd` 拖入文件夹 → 回车 → `npm run dev`
2. 终端 2（新窗口）：`cd` 拖入文件夹 → 回车 → `cd mcp` → `npm run receiver`
3. CodeBuddy 预览打开 `http://localhost:5173/` → 拖拽 → 同步 MCP → 对话让 AI 落实

**两个终端窗口全程都要开着，别关。**

## 给管理员的话（同事不用看）

为降低同事门槛，分发前请**预先做好这两件事**，可省掉同事最容易出错的步骤：

1. **提前改好 `index.html`**：把 `init()` 改成 `init({ enableMcpSync: true })` 并保存/提交。这样同事完全不用碰代码。
2. **提前跑一次 `npm install`（含 `mcp/` 子目录）**，把 `node_modules` 一起打包给同事，同事就能跳过【步骤 5】。
3. 告诉同事 CodeBuddy 里的 MCP 是否已配置好——如果每台电脑路径不同，需要在各自的 `~/.codebuddy/mcp.json` 里把 `server.mjs` 路径改成该电脑上的实际路径（可用「拖拽文件到终端得到路径」的方法获取）。
