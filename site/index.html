How it works

三步，从「感觉不对」到「已经改好」

不用截图圈红框，不用打字描述「往左一点、再小一点」。

01

直接改到满意

点选元素拖到位、拉到合适大小，在属性面板里调间距、颜色、圆角，双击改文案。

02

自动记录 diff

每处修改生成稳定选择器 + before/after 盒模型 + 样式变更，附前后快照缩略图。

03

交给 AI 或复制 CSS

一键复制结构化 Markdown 粘给 coding agent 改真实源码；或直接导出可用的 CSS。

复制 Markdown 后你会得到
# Sidetation 视觉修改记录（2 处）

## 1. `div.cta-group > button.btn-primary`
- 移动：Δx +24px，Δy 0px
- 尺寸：112×47 → 160×56
- 样式：`border-radius: 12px`，`background-color: #3b6ef6`

## 2. `section.hero > h1`
- 文字："旧标题" → "新标题"

请把以上视觉改动落实到源码中：根据元素所在布局
选择合适的方式（margin / flex / gap / width …）
Features

为「所见即所得的反馈」而生

全部功能零依赖、框架无关，注入任何页面即可使用。

🖐️

拖拽移动

transform 位移不破坏文档流，Figma 式参考线自动吸附对齐。

↔️

手柄缩放

8 向手柄，Shift 等比，content-box 自动补偿，对侧边缘保持固定。

🎛️

属性面板

Auto layout（flex 方向 / 对齐九宫格 / gap / 四边 padding）、Fill、Stroke、圆角、透明度。

✏️

文字编辑

双击纯文本元素行内编辑，纯文本粘贴，改动同样进入修改记录。

🖼️

前后快照

每条修改附「优化前 → 优化后」缩略图，原生 SVG 实现，无第三方依赖。

⌘

完整撤销栈

手势级 undo / redo，连续微调自动合并，Delete 删除也可一键恢复。

🎯

稳定选择器

优先 id / data-testid，过滤 CSS-in-JS 哈希类名，AI 能直接 grep 到源码。

📋

双格式导出

结构化 Markdown 给 coding agent；或复制可直接粘贴的 CSS 覆盖。

🧩

多选批量编辑即将上线

Shift 加选多个元素，整组拖动、删除、改样式，一步撤销。

Install

四种方式，10 秒用上

自己的项目用 npm 包；别人的网站用 Chrome 插件、控制台粘贴或书签注入。

npm 包（推荐，开发环境）

零依赖、框架无关，React / Vue / 原生页面都能用。

npm install sidetation
import { init } from 'sidetation';

if (import.meta.env.DEV) init();

npm 包正在发布中；发布前可用下方的 GitHub 构建方式提前体验。

Chrome 插件（任意网站，推荐给团队）

点击工具栏图标即可在当前页开启 / 关闭编辑模式，带 ON 状态徽标。仅申请 activeTab 权限——只在你主动点击时注入当前页，无「读取所有网站」类敏感权限。

git clone https://github.com/tllll64/Sidetation.git && cd Sidetation && npm install && npm run build

构建后打开 chrome://extensions → 开启「开发者模式」→「加载已解压的扩展程序」→ 选择仓库里的 extension/ 目录即可。Chrome Web Store 上架版本筹备中。

控制台粘贴 / 书签注入（任意网站）

从 GitHub 克隆并构建一次，得到两个注入文件：dist/console-paste.js 粘进任意网站的 DevTools 控制台即可；dist/bookmarklet.txt 存为书签则可一键注入。控制台方式不受页面 CSP 限制。

git clone https://github.com/tllll64/Sidetation.git && cd Sidetation && npm install && npm run build

构建完成后打开 dist/console-paste.js，全选复制，到目标网站按 F12 打开控制台粘贴回车即可。

FAQ

常见问题

它和 agentation 有什么区别？

agentation 的思路是「点选元素 + 文字标注」，Sidetation 把表达方式升级为「直接操纵」：拖到位、拉到大小、改到颜色，工具负责把这些动作翻译成结构化 diff。两者殊途同归——都不生成代码，只生成让 AI 精确定位的上下文。

拖拽会破坏页面布局吗？

不会。移动用 transform: translate 实现，不触发重排；所有修改只写内联样式并保存原值，撤销即完整还原，刷新页面即全部消失。

导出的 Markdown 为什么不直接给 CSS？

给 AI 的记录会刻意标注「不要照抄 transform」——位移只是编辑器里的临时表现，AI 应该根据元素所在布局改 margin / flex / gap 等真正的布局属性。想要立即生效的样式，用「复制 CSS」。

在别人的网站上能用吗？

能。控制台粘贴或书签注入即可，UI 渲染在 Shadow DOM 隔离层里不受宿主样式影响，也兼容开启 Trusted Types 的站点。注意：第三方网站上导出的选择器对应不到你的源码，主要用「复制 CSS」。

支持哪些快捷键？

与 Figma 对齐：方向键微调（Shift ×10）、Delete 删除、⌘Z / ⌘⇧Z 撤销重做、Enter / Shift+Enter 下钻上钻、Tab 切换兄弟元素、双击改文字、Esc 退出。编辑模式下工具栏的 ⌘ 按钮可查看完整列表。
